# 一致性检查规则

## 防止的问题

### 1. 幻视（前后矛盾）
- **定义**：前后文出现矛盾，设定不一致
- **检查项**：
  - 角色设定前后是否一致
  - 世界观设定是否一致
  - 力量体系是否一致
  - 时间线是否一致
  - 地点是否一致

### 2. 莫名其妙的编造
- **定义**：突然出现没有铺垫的内容
- **检查项**：
  - 新角色出现是否有铺垫
  - 新能力出现是否有说明
  - 新设定出现是否有解释
  - 情节转折是否有伏笔

### 3. 牛头不对马嘴
- **定义**：内容与上下文不符
- **检查项**：
  - 对话是否符合人物性格
  - 行为是否符合人物设定
  - 情节是否符合世界观
  - 能力使用是否符合设定

### 4. 出现"章节"等字样
- **定义**：正文中出现"第X章"、"本章"等字样
- **检查项**：
  - 正文中不能出现"章节"字样
  - 正文中不能出现"本章"字样
  - 正文中不能出现章节编号
  - 正文中不能出现"上一章"、"下一章"等

### 5. 时间线混乱
- **定义**：时间顺序错误或混乱
- **检查项**：
  - 事件发生顺序是否合理
  - 时间跨度是否合理
  - 时间描述是否一致
  - 时间跳跃是否有说明

## 检查方法

### 自动检查（规则引擎）
1. **关键词检查**：
   - 检查是否出现"章节"、"本章"等字样
   - 检查是否出现章节编号

2. **时间线检查**：
   - 提取时间相关词汇
   - 检查时间顺序

3. **设定检查**：
   - 对比memory文件中的设定
   - 检查是否出现新设定

### LLM检查
1. **一致性检查**：
   - 对比当前章节与memory文件
   - 检查设定是否一致

2. **逻辑检查**：
   - 检查情节逻辑
   - 检查人物行为逻辑

3. **铺垫检查**：
   - 检查新内容是否有铺垫
   - 检查伏笔是否合理

## 修复方法

### 自动修复
1. **删除"章节"字样**：自动删除正文中的"章节"等字样
2. **修正时间线**：根据上下文修正时间顺序
3. **补充铺垫**：为新内容添加铺垫

### 人工修复
1. **设定冲突**：需要人工判断和修复
2. **逻辑问题**：需要人工分析和修复
3. **铺垫不足**：需要人工补充铺垫

## 预防措施

### 生成前
1. **读取所有memory文件**：确保了解所有设定
2. **检查时间线**：确认当前时间点
3. **检查设定**：确认世界观、力量体系等

### 生成中
1. **实时检查**：生成过程中检查一致性
2. **引用设定**：明确引用memory文件中的设定
3. **避免创新**：不随意添加新设定

### 生成后
1. **全面检查**：检查所有一致性项
2. **对比memory**：与memory文件对比
3. **修复问题**：修复发现的问题

## 检查清单

每次生成后必须检查：
- [ ] 是否出现"章节"等字样
- [ ] 时间线是否一致
- [ ] 设定是否一致
- [ ] 人物行为是否一致
- [ ] 能力使用是否一致
- [ ] 地点是否一致
- [ ] 新内容是否有铺垫
- [ ] 伏笔是否合理
- [ ] 逻辑是否合理
- [ ] 前后文是否连贯
